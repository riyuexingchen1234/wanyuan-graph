## 第一部分 · 审核员的角色与责任

### 1.1 角色定义

万源图谱采用 **80/15/5 生产流程**：

| 环节 | 比例 | 责任主体 | 产出 |
|---|---|---|---|
| 候选边搜集 | 80% | AI Agent（按 80% 通过率过滤明显无据） | `verification_status: proposed` |
| 复核 | 15% | 编辑团队（事实校对、表述规范） | `verification_status: verified`（**编辑级**，仅完成证据自洽性核对） |
| **终审** | **5%** | **审核员（行业专家 / 资质人员）** | `verification_status: verified` + `reviewed_by` + `reviewed_at`（`verified-expert` 状态） |

审核员位于流程末端，其签名的边是图谱**权威性的根基**：每一条 `verified-expert` 状态的边都可追溯到具体审核员。`schema.json` v0.3 已为此新增 `reviewed_by` 与 `reviewed_at` 字段，并明确**"AI 不得填写此字段"**。

### 1.2 核心责任

1. **真伪判断**：对提交到面前的边，确认其在现实世界中是否真实存在。
2. **evidence 链回溯**：评估 evidence 是否能回溯到一级源（行业标准 / 国家标准 / 官方数据 / 上市公司公告 / 专利 / 政府招投标公示等）。
3. **明确拒绝**：对不合格的边，打回并**写明具体理由**（关系不成立 / 类型不准确 / 方向反 / 粒度不对），不是含糊的"再看看"。
4. **主动降级**：对已 verified 的边，如发现新证据表明其不再成立（如标准更新、企业产品停产、关系被证伪），有权将 `verified` 降级为 `disputed`（schema 当前为 `proposed`；待 v0.4 引入 `disputed` 状态后正式生效——见"待用户确认项 #3"）。

### 1.3 不负责什么（边界，重要！）

| 审核员**不**做 | 由谁做 | 备注 |
|---|---|---|
| 价值判断（"这个产业值不值得投""这个材料好不好用"） | 分支产品 / 用户自己 | 万源图谱不背书任何商业判断，审核员也不背书 |
| 补全信息（节点缺 definition / 边缺 evidence） | 提案者 / 编辑团队 | 审核员代写会污染"判断权" |
| 修改节点本体（name / definition / parent_type） | 提案者 / 编辑团队 | 节点的术语与分类是提案者负责，审核员只审"边" |
| 给边加"机会评分"或"重要性"权重 | 留待产品层 | `note` 字段允许客观说明，不允许价值描述 |
| 与其他审核员"合议"后给一致结论 | 各自独立判断 | 见"第六部分 守则" |

## 第二部分 · 审核员资质要求

### 2.1 基础资质（任一即可）

1. 行业从业 **5 年以上**（在职证明 / 离职证明 / 职业证书任一）。
2. 相关学科 **硕士及以上** 学位（材料 / 化工 / 物理 / 产业经济 / 冶金 / 半导体 / 能源工程等）。
3. 行业协会注册 **工程师 / 高级工程师**（中国电子学会、中国化工学会、中国材料研究学会、中国光伏行业协会 CPIA、中国化学与物理电源行业协会 CIAPS 等）。
4. 上市公司**技术高管**（在任或曾任 CTO / 技术总监 / 总工程师 / 研发负责人）。
5. **国家标准 / 行业标准 / 团体标准起草人**（标准文本署名页可查证）。

### 2.2 资质核实流程

```
提交材料 → 平台运营方交叉验证 → 录入审核员档案 → 签发审核资格
```

- **提交材料**：学位证书 / 职业证书 / 在职证明 / 标准署名页 / 公开履历截图。
- **交叉验证渠道**（任一即可）：
  - 国家政务服务平台 / 学信网
  - 全国社会组织信用信息公示平台（行业协会真实性）
  - 天眼查 / 启信宝（企业任职）
  - 全国标准信息公共服务平台 standard.samr.gov.cn（标准署名）
  - 巨潮资讯网 / 上交所 / 深交所披露的年报（高管理事）
- **有效期**：2 年；到期前 30 天系统提醒，提交近一年新材料即可复审。

### 2.3 专业领域声明

审核员**必须**在档案中声明自己擅长的产业 / 技术领域。平台分配边时按"领域匹配度"过滤，不匹配的边不进入队列。

| 声明示例 | 匹配边 | 不匹配边（被过滤） |
|---|---|---|
| "光伏 + 半导体电子封装" | 银浆、电池片、TOPCon 工艺、HJT、MLCC 浆料 | 锂电池正极材料、煤化工 |
| "锂离子电池 + 储能" | 锂矿、碳酸锂、电解液、隔膜 | 光伏组件、风电叶片 |
| "高分子材料 + 包装" | 聚乙烯、EVA 胶膜、塑料管道 | 金属粉末冶金、稀土 |

**例**：A 审核员只声明了"光伏 + 半导体"，则 `锂电池正极材料 → 隔膜 consumable_for` 这类边不会出现在 A 的待审队列。

### 2.4 利益冲突回避

| 冲突类型 | 必须回避的情形 |
|---|---|
| **所在公司** | 审核员所在公司及其**控股 / 持股 5% 以上**关联公司涉及的边 |
| **直系亲属** | 配偶 / 父母 / 子女及其配偶所在公司涉及的边 |
| **直接竞品** | 同一细分赛道（市占率前三）直接竞争对手的边 |
| **客户 / 供应商** | 审核员公司是该边任一节点的客户或主供应商时 |

**案例**：在某光伏组件厂（假设隆基绿能）任职的工程师，**不**审核另一光伏组件厂（晶科能源）直接竞品相关的边，如"晶科能源 TOPCon 电池 → 组件 can_be_processed_into"。但**可以**审核"多晶硅 → 硅片 upstream_of"这类通用边。

冲突申报**主动发起**：接到待审边时，**第一步**就自查利益冲突；如发现冲突，在 4 小时内点"转派"按钮，不进入实质审核。

## 第三部分 · 审核流程标准操作程序（SOP）

### 3.1 接收任务

平台从复核队列向审核员分配一条边（含 `id`、`source`、`target`、`relation_type`、`evidence`、`proposed_by`）。审核员有 **72 小时**响应：

- **接受**：进入审前自查。
- **转派**：声明理由（领域不匹配 / 利益冲突 / 时间不足），退回平台重派。

### 3.2 审前自查清单（4 项必查）

- [ ] **我是否在擅长领域内？** —— 超出领域立即转派。
- [ ] **是否存在利益冲突？** —— 存在立即转派。
- [ ] **evidence 是否能点击 / 检索到？** —— 失效或不可访问则打回。
- [ ] **source 和 target 节点的定义我是否理解准确？** —— 模糊则查阅节点本体；定义本身就是"什么"而非"是否"的问题。

### 3.3 核心审核标准（4 个决策点）

#### 决策点 ①：关系是否存在？

> 边声明的连接在现实中是否成立？

**审核方法**：
1. 翻阅 evidence 给出的标准 / 公告 / 专利。
2. 必要时补充检索（一级源优先）。
3. 至少 **1 条 evidence** 回溯到一级源（标准 / 专利 / 官方数据 / 上市公司公告）。

**判断示例**（来自真实公开资料）：
- 待审边：`硝酸银 → 银粉 raw_material_for`
- 证据：液相化学还原法（公开学术文献 + 行业研报可查）证实硝酸银是制备光伏银粉的标准前驱体；帝科股份 2022 年公告（巨潮资讯网 2022-077）也明确规划"年产 5000 吨硝酸银"作为公司**银粉自供**的上游配套项目。
- 决策：**通过**（关系在产业实践中真实存在）。

#### 决策点 ②：关系类型是否准确？

> `relation_type` 选对了没有？参考 `schema.json` 中的优先级规则：
>
> "raw_material_for / equipment_for / consumable_for 等细分类型应优先使用；upstream_of / downstream_of 是泛化的物理流动兜底类型，仅在一条关系明确属于物理流动性质、但又不属于任何已有细分类型时使用。"

**判断示例**：
- 待审边：`银浆 → 电池片 consumable_for`
- 候选类型对比：

  | 类型 | 含义 | 适用性 |
  |---|---|---|
  | raw_material_for | 作为原料供给（构成主体的成分） | ❌ 银浆不是构成电池片的成分（银浆烧结后形成电极，电极是电池的一部分，但银浆本身经烧结后结构改变） |
  | consumable_for | 作为耗材供给（生产中消耗） | ✅ 银浆在丝网印刷 + 烧结后转变为电极栅线本身，浆料状态消失，是典型的"持续消耗的辅材" |
  | applied_in | 被用于某应用 / 场景 | ✅ 也可接受，但 consumable_for 更精确 |

- 决策：保留 `consumable_for`，通过。

#### 决策点 ③：方向是否正确？

> source 和 target 是否写反？

**反例**：
- `电池片 → 银浆 raw_material_for` —— **反了**。银浆是投入到电池片生产中的，方向应是 `银浆 → 电池片`。
- `多晶硅 ← 工业硅 upstream_of` —— **反了**。应是 `工业硅 → 多晶硅`（上游 → 下游）。

判断技巧：先写下"X 是 Y 的原料/耗材"这句话，如果读起来不自然，方向大概率反了。

#### 决策点 ④：粒度是否合适？

> 节点是过粗还是过细？

**过粗反例**：
- "电池片"应细化为"晶硅电池片"（区分单晶 / 多晶 / TOPCon / HJT）。
- "组件"应明确"光伏组件"，并区分是否包含电池片（光伏组件 = 电池片 + 玻璃 + 背板 + 边框 + 接线盒 + 封装胶膜）。

**过细反例**：
- 把"光伏组件"拆成"主栅银浆"+"副栅银浆"+"铝浆"等十几条边，把浆料族拆得过碎，影响图谱可读性。**粒度统一原则**：粒度应与同层节点相当，避免出现"原材料 + 零部件 + 完整产品"混在同一深度的混乱局面。

### 3.4 决策选项（必须四选一）

| 决策 | 何时使用 | 必填字段 |
|---|---|---|
| **通过** | 边合格，升级为 `verified-expert` | `reviewed_by`（审核员唯一标识）、`reviewed_at`（ISO 8601） |
| **打回** | 边不合格，返 `proposed` 队列 | `reviewed_by`、`reviewed_at`、`note` 注明理由（"关系不成立"/"类型不对"/"方向反"/"粒度不对"四选一或组合） |
| **降级** | 已 `verified` 边被新证据推翻 | `reviewed_by`、`reviewed_at`、`note` 说明新证据（schema v0.4 引入 `disputed` 后生效） |
| **合并 / 拆分** | 建议与已 verified 边合并 / 拆成多条更细边 | `reviewed_by`、`reviewed_at`、`note` 说明合并 / 拆分对象与原因；**不直接修改**，需另开编辑流程 |

### 3.5 审核留痕要求（不可涂改）

```json
{
  "id": "edge-silver-powder-to-silver-paste",
  "source": "material-silver-powder",
  "target": "material-silver-paste",
  "relation_type": "made_of",
  "verification_status": "verified",
  "evidence": [ ... ],
  "reviewed_by": "reviewer-zhang-2026-001",
  "reviewed_at": "2026-06-28T10:23:45+08:00",
  "note": "聚和材料 2023 年报第二节明确'光伏导电银浆是一种以银粉为基材的功能性材料'，银粉占银浆原材料成本 90% 以上。relation_type 选 made_of，反映银粉是银浆的构成主体。"
}
```

| 字段 | 必填 | 格式 | 说明 |
|---|---|---|---|
| `reviewed_by` | ✓（verified 边） | 平台分配的稳定唯一标识 | **不可直接修改**；纠错必须通过"再次审核"流程覆盖 |
| `reviewed_at` | ✓（verified 边） | ISO 8601，含时区 | 决策时间，不接受模糊的"今天" |
| `note` | ✗（建议填） | 1-2 句客观陈述 | **禁止**价值判断（"很重要""值得推广"等） |

### 3.6 决策树（一图总览）

```
接收边
  │
  ├─ 72h 内转派？─是→ 平台重派（结束）
  │
  ├─ 审前自查 4 项
  │   ├─ 领域不匹配 / 利益冲突 / 证据失效 → 转派或打回
  │   └─ 全部通过
  │
  ├─ 决策点 ① 关系是否存在？─否→ 打回 "关系不成立"
  │
  ├─ 决策点 ② 关系类型是否准确？─否→ 打回 "类型不对"
  │
  ├─ 决策点 ③ 方向是否正确？─否→ 打回 "方向反"
  │
  ├─ 决策点 ④ 粒度是否合适？─否→ 打回 "粒度不对"或建议合并/拆分
  │
  └─ 全部通过 → 填写 reviewed_by / reviewed_at / note → 升级 verified-expert
```

## 第四部分 · 典型场景案例库

> 以下案例基于**真实公开资料**撰写（行业标准、上市公司公告与年报、学术文献）。审核员在具体审核时仍须以原文为准；本节用于训练判断力，不是替代审核。

### 案例 1：通过案例 —— `硝酸银 → 银粉 raw_material_for`

**待审边完整内容**：

| 字段 | 值 |
|---|---|
| `source` | material-silver-nitrate（硝酸银） |
| `target` | material-silver-powder（银粉） |
| `relation_type` | raw_material_for |
| `evidence[0]` | `T/CSEA` 系列行业标准（硝酸银产品标准，**具体编号待审核员在标准库中检索确认**——见"待用户确认项 #1"） |
| `evidence[1]` | 帝科股份 2022-077 公告（巨潮资讯网 2022-07-29），披露公司规划"年产 5000 吨硝酸银"项目，并说明其"作为含银金属粉体生产的更上游原材料" |
| `proposed_by` | editorial_research（编辑团队研究提出） |

**审核员检查过程**：
1. 查 ② 类型：`raw_material_for`（作为原料供给）。硝酸银经化学还原后**直接**生成银粉，是典型的原料关系，类型准确。
2. 查 ③ 方向：先写"硝酸银是银粉的原料"——自然，方向正确。
3. 查 ① 关系是否存在：液相化学还原法是行业标准工艺（公开学术文献《粉末冶金工业》2025 年 35(2): 114-118 详细描述"以硝酸银为银源、抗坏血酸为还原剂"制备光伏银粉）；帝科股份公告也直接说明硝酸银 → 金属粉的供应链关系。关系成立。
4. 查 ④ 粒度：硝酸银和银粉是产业实践中通用的两个层级，粒度合理。

**最终决策**：**通过**，升级为 `verified-expert`。

**留痕记录**：

```json
{
  "reviewed_by": "reviewer-zhang-2026-001",
  "reviewed_at": "2026-06-28T10:23:45+08:00",
  "note": "液相化学还原法是行业通用工艺，帝科股份 2022-077 公告明确硝酸银为银粉自供的上游原材料，关系成立、类型与方向均无误。"
}
```

### 案例 2：通过案例 —— `银粉 → 银浆 made_of`

**待审边完整内容**：

| 字段 | 值 |
|---|---|
| `source` | material-silver-powder（银粉） |
| `target` | material-silver-paste（银浆） |
| `relation_type` | made_of |
| `evidence[0]` | 聚和材料（688503.SH）2023 年年度报告（上交所披露 2024-04-01） |

**审核员检查过程**：
1. 查 ② 类型：银粉是银浆的构成主体（含量占银浆原材料成本 **90% 以上**），用 `made_of`（由某材料构成）准确，比 `consumable_for` 更精准。
2. 查 ① 关系是否存在：聚和材料 2023 年报第二节"主要业务介绍"原文表述"光伏导电银浆是一种以银粉为基材的功能性材料"，**直接**给出"银粉 → 银浆"的"主料—制品"关系。
3. 查 ③ 方向：先写"银粉构成银浆"——自然，方向正确。
4. 查 ④ 粒度：同层节点均为"material-xxx"，粒度统一。

**最终决策**：**通过**。

**留痕记录**：

```json
{
  "reviewed_by": "reviewer-li-2026-014",
  "reviewed_at": "2026-06-28T14:08:12+08:00",
  "note": "聚和材料 2023 年报原文'光伏导电银浆是一种以银粉为基材的功能性材料'，银粉占银浆原材料成本 90% 以上，made_of 关系成立。"
}
```

### 案例 3：通过案例 —— `银浆 → 电池片 consumable_for`

**待审边完整内容**：

| 字段 | 值 |
|---|---|
| `source` | material-silver-paste（银浆） |
| `target` | product-pv-cell（光伏电池片） |
| `relation_type` | consumable_for |
| `evidence[0]` | T/CPIA 0030 系列团体标准——晶体硅光伏电池用浆料（中国光伏行业协会发布） |

**审核员检查过程**：
1. **核对标准号**：T/CPIA 0030 是**系列标准**（不是单条标准），由 CPIA 发布，包括：
   - T/CPIA 0030.1 背场铝浆 烧结型铝浆
   - T/CPIA 0030.2 背面银浆 烧结型银浆（2021 年发布；2025 年发布 0030.2-2025 修订版）
   - T/CPIA 0030.3 正面银浆 烧结型银浆（2021 年发布；2025 年发布 0030.3-2025 修订版）
   - T/CPIA 0030.4 正/背面银浆 固化型银浆（2024 年发布）
   - 审核员应**写明**具体引用的是 .2、.3 还是 .4 部分，**不接受**只写"T/CPIA 0030"的模糊引用。
2. 查 ② 类型：参考 `DB33/T 1322-2023《产业链图谱构建指南》` 第 4.3 节表 1（产业链图谱的上下游关系类型），其将"**辅助材料**"定义为"主体应用时的配套材料，或用于让主要材料发生变化的材料，或主体完工前应使用的其他非生产原料类材料"。银浆在丝网印刷 + 烧结后浆料状态消失、转变为电池片电极栅线的一部分，与 DB33 标准中"用于让主要材料发生变化的材料"高度吻合，对应 schema 体系中的 `consumable_for`（作为耗材供给）。
3. 查 ① 关系是否存在：T/CPIA 0030 系列标准本身就是为"晶体硅光伏电池用浆料"制定的，**直接**确认银浆对电池片的功能性供给。
4. 查 ③ 方向：先写"银浆是电池片生产的耗材"——自然，方向正确。
5. 查 ④ 粒度：源节点（material-银浆）与目标节点（product-电池片）跨 material → product 类型，**粒度层级合理**。

**最终决策**：**通过**，但要求提案者补充具体的 T/CPIA 0030 子编号。

**留痕记录**：

```json
{
  "reviewed_by": "reviewer-wang-2026-007",
  "reviewed_at": "2026-06-28T16:42:08+08:00",
  "note": "T/CPIA 0030 系列标准直接定义银浆为'晶体硅光伏电池用浆料'，结合 DB33/T 1322-2023 第 4.3 节表 1'辅助材料'定义，consumable_for 类型准确。已要求补充具体子标准号（0030.2 / 0030.3 / 0030.4）。"
}
```

### 案例 4：打回案例 —— `硅片 → 银浆 raw_material_for`（方向反 + 银浆是辅料非主料）

**待审边完整内容**：

| 字段 | 值 |
|---|---|
| `source` | product-silicon-wafer（硅片） |
| `target` | material-silver-paste（银浆） |
| `relation_type` | raw_material_for |
| `evidence` | 仅有研报转述，无原始标准 / 公告 / 专利 |

**审核员检查过程**：
1. 查 ③ 方向：先写"硅片是银浆的原料"——**不自然**。银浆不来源于硅片，方向反了。正确方向应是 `银浆 → 硅片 consumable_for`（银浆印刷在硅片表面）。
2. 查 ② 类型：即便方向修正，`raw_material_for`（构成主体的成分）也不准确。硅片不是由银浆构成。正确的是 `consumable_for` 或 `applied_in`。
3. 查 evidence：仅有研报转述（**二三级来源**），未回溯到一级源。

**最终决策**：**打回**。

**留痕记录**：

```json
{
  "reviewed_by": "reviewer-zhang-2026-001",
  "reviewed_at": "2026-06-28T11:05:30+08:00",
  "note": "打回原因：1) 方向反，硅片不是银浆的原料；2) 类型不对，银浆不是硅片的构成主体，应为 consumable_for/applied_in；3) evidence 未回溯到一级源，建议补充 T/CPIA 0030 子标准或电池厂工艺文件。请提案者修正后重新提交。"
}
```

### 案例 5：降级案例 —— 已 verified 边因标准更新被降级

**背景**：
一条已 `verified-expert` 的边 `烧结型银浆 → 异质结电池片 consumable_for`，2024 年审核通过，evidence 引用 T/CPIA 0030.2-2021 背面银浆烧结型银浆。2025 年 3 月，T/CPIA 0030.2-2025 修订版发布，新版标准明确将"适用于异质结电池的烧结型银浆"移出适用范围（HJT 需用低温固化型银浆，对应 T/CPIA 0030.4 系列）。

**审核员检查过程**：
1. 检索证据：发现 T/CPIA 0030.2-2025（中国光伏行业协会 2025-03-27 标准公告）已正式发布，且明确界定适用范围。
2. 核对原 evidence：原 evidence 引用的 2021 版标准不再覆盖 HJT 电池。
3. 评估影响：原"烧结型银浆 → 异质结电池片 consumable_for"边在新标准下**不再成立**。

**最终决策**：**降级**（待 schema v0.4 引入 `disputed` 状态后生效——见"待用户确认项 #3"）。当前流程：在 note 中标注"建议降级，待 v0.4 上线后由平台批量迁移"。

**留痕记录**：

## 第五部分 · 常见问题 FAQ

**Q1：evidence 链接失效了怎么办？**
A：分两种情况。
- 链接**本身**失效但内容**可查证**（如标准号、年报期次、专利号仍可定位）：不一定要打回，但 note 中标注"链接已失效，但可凭 XXXX 标准号 / 公告号在 [具体平台] 查到"——这本身是 `source_type` 应记录的内容，不是 evidence 失效。
- 内容**不可查证**（原始来源彻底消失或被覆盖）：打回，要求提案者补充可查证来源。

**Q2：源文本与边声明不完全一致（措辞差异）怎么办？**
A：判断"实质内容"是否一致，不是字面抠词。例如源文说"光伏导电银浆是制备太阳能电池金属电极的关键材料"，边声明"银浆 → 电池片 consumable_for"——文字不同但**实质相同**（金属电极就是电池片的一部分），通过。反例：源文说"银浆可用于……（列举 5 个场景）"但边声明只覆盖其中 1 个场景，**仍可接受**（边只声明存在的连接，不声明唯一的连接）。

**Q3：同一关系有多个权威源但表述略不同，选哪个为准？**
A：优先级 = 一级源 > 二级源 > 三级源；同级别时优先**可验证性最强**的（标准号 / 公告号 / 专利号 > 文件标题）。建议 evidence 数组**同时**列出多源（一级 + 一级，或一级 + 交叉印证的二级），让用户自己看。

**Q4：我想修改 evidence 内容，可以吗？**
A：审核员**不**直接编辑 evidence（"第六部分 守则"明文禁止代写）。如果发现 evidence 引用错误，**打回**让提案者修正；如果是审核过程中自己检索到新证据，可在 note 中**引用**新证据的来源供提案者参考，但仍由提案者更新 evidence 字段。

**Q5：用户 / 贡献者对我审核结果不服，怎么办？**
A：标准流程：
1. 贡献者可在平台上**申诉**，附新证据。
2. 申诉进入**仲裁队列**，由另一名（不重叠领域的）审核员或编辑委员会复审。
3. 仲裁结果为终局。
审核员**不得**与申诉者私下沟通、修改结论。如发现自己错了，正确做法是走"再次审核"流程（用新一条 reviewed_by 记录覆盖旧记录），而不是悄悄修改原 `reviewed_by`。

**Q6：我可以撤回自己的审核吗？**
A：可以。撤回 = 相当于"打回 + 转交其他审核员"，会保留原 `reviewed_by` / `reviewed_at` 记录并新增一条"撤回"日志（`note` 写"撤回原因"），**不删除原记录**，确保全程可追溯。

## 第六部分 · 审核员守则

1. **不可代写 evidence** —— 哪怕你认为原文不全、不准确。修改 evidence 是提案者 / 编辑团队的工作，审核员只判断已有 evidence 是否充分。
2. **不可接受利益相关方的咨询费 / 感谢费 / 馈赠** —— 一经发现立即取消审核资格。
3. **不可与其他审核员私下合议** —— 各自独立判断，意见不一致由仲裁流程决定，不通过"讨论"形成默契结论。
4. **主动申报利益冲突** —— 接到任务第一时间自查，4 小时内转派。
5. **不可给价值判断** —— `note` 中**不**写"很重要""值得推广""可能是下一个风口"等表述，只写"evidence 来源是 XXXX，关系成立 / 不成立的依据是 XXXX"。
6. **不可直接修改历史记录** —— 纠错走"再次审核"流程覆盖，不动原 `reviewed_by` / `reviewed_at`。
7. **不可泄露未公开边** —— 审核中的边是内部资料，未 `verified-expert` 状态的不对外发布。

## 附录 A · 关系类型速查表（对照 `schema.json` v0.3）

| relation_type | 中文释义 | 何时使用 |
|---|---|---|
| `raw_material_for` | 作为原料供给 | 构成主体成分（X 是 Y 的基本材料） |
| `equipment_for` | 作为设备供给 | 用于生产的设备（X 是 Y 用的机器） |
| `consumable_for` | 作为耗材供给 | 生产中持续消耗、不构成主体（X 在 Y 生产中被消耗） |
| `can_be_processed_into` | 材料经工艺产出某物 | 加工转化（X 经工艺生成 Y） |
| `applied_in` | 被用于某应用 / 场景 | 场景性使用（X 用于 Y） |
| `structurally_similar_to` | 横向相似性 | 同层跳转（X 与 Y 在属性上相似） |
| `made_of` | 由某材料构成 | 主体由材料构成（X 是 Y 的组成） |
| `upstream_of` / `downstream_of` | 泛化上下游（兜底） | 仅在细分类型均不适用时使用 |
| `satisfies` | 满足需求 | 节点满足某需求节点（v0.3 新增） |

## 待用户确认项清单

1. **硝酸银 / 银粉对应的中国行业标准具体编号**：本手册引用 T/CSEA 系列作为"硝酸银产品标准"证据，但**未**在公开检索中核到 T/CSEA 专门针对"硝酸银 → 银粉"工艺的标准编号。需用户在国标库 / 行业标准库进一步确认具体编号（或确认是否改用其他一级源）。
2. **审核员唯一标识 `reviewed_by` 的编码规则**：手册示例用 `reviewer-姓名拼音-年份-序号`，但平台是否已有定式？建议用户在 v0.1 评审时明确（是否含真实姓名？是否脱敏？）。
3. **schema 引入 `disputed` 状态的时间表**：案例 5 涉及"降级"动作，但 schema.json v0.3 仅有 `verified` / `proposed` 两态。建议用户确认 v0.4 引入 `disputed` 的时间点，以及在过渡期内"降级"的实际处理方式（暂存 note？先转 proposed？）。
4. **利益冲突判定的"5% 持股"阈值**：是否参照证券法的"5% 举牌线"？或采用更严格 / 宽松的阈值？
5. **多审核员仲裁的具体流程**：FAQ Q5 提到"仲裁队列由另一名审核员复审"，但未明确仲裁员是否匿名、是否需要 3 人合议、是否有终审委员会。需用户在 v0.2 增补。
6. **`T/CPIA 0030` 引用规范**：案例 3 已说明"必须写明 .2 / .3 / .4 子编号"，是否需要在审核员 UI 强制必填子编号？建议在 v0.2 增加前端校验。
7. **审核员工作量上限**：72 小时响应是基础要求，但每月 / 每周最多审多少条边以保证质量？是否需要设定上限？建议用户确认。
8. **案例中研报 / 学术文献的引用强度**：案例 1 的化学还原法引用了《粉末冶金工业》论文（2025）作为"液相化学还原"工艺的一级源，但严格来说**学术论文**介于一级与二级之间。需用户确认此类来源在审核中的接受标准（"被引次数"？期刊等级？）。
