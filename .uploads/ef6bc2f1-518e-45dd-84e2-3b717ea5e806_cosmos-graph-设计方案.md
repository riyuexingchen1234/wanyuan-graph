# Cosmos Graph 3D 产业链宇宙 · 设计方案

> 三态交互系统：星云探索 → 飞渡过渡 → 焦点导航
> 核心体验：宇宙感 + 探索欲 + 「抵达」的仪式感

---

## 一、整体架构

系统有**三种状态**，通过用户交互切换：

```
┌─────────────┐   点击节点/搜索    ┌─────────────┐   动画完成    ┌─────────────┐
│  ① 星云探索态 │ ────────────────→ │  ② 飞渡过渡态 │ ────────────→ │  ③ 焦点导航态 │
│   Nebula    │                    │   Flying    │               │    Focus    │
└─────────────┘                    └─────────────┘               └──────┬──────┘
      ↑                                    ↑                            │
      │         点击「返回星云」(可选)       │     点击其他节点（重排）     │
      └────────────────────────────────────┴────────────────────────────┘
                                       点击祖先节点：仅相机复位，不重排
```

## 二、状态一：星云探索态（初始状态）

### 2.1 视觉表现

- 相机静止在 `(0, 5, 100)`，看向星云中心 `(0, 0, -120)`
- 所有节点在远处以不同大小的椭圆轨道缓慢公转
- 节点呈星点状（小圆球 + 微弱光晕），连线极淡或不可见
- 背景是 3000+ 颗星点构成的球形星空（500 半径球壳）
- 营造「漂浮在宇宙中俯瞰星河」的第一印象

### 2.2 椭圆轨道模型

每个节点有独立的随机轨道参数：

| 参数 | 范围 | 说明 |
|------|------|------|
| 长轴 a | 60 ~ 140 | 椭圆水平半径 |
| 短轴 b | 30 ~ 80 | 椭圆垂直半径 |
| 公转速度 ω | 0.05 ~ 0.15 rad/s | 缓慢、各节点速度不同 |
| 轨道倾斜 tilt | ±0.3 rad | 每个轨道有随机倾斜，避免共面 |
| 水平旋转 rot | 0 ~ 2π | 椭圆主轴方向随机 |
| 初始相位 phase | 0 ~ 2π | 初始角位置随机 |
| Y 轴压缩 | ×0.4 | 让轨道呈盘状（扁椭圆），更像星系 |

公转更新公式（每帧）：

```
angle = time * ω + phase
x_local = a * cos(angle)
z_local = b * sin(angle)
// 绕 X 轴倾斜
y1 = y*cos(tilt) - z*sin(tilt)
z1 = y*sin(tilt) + z*cos(tilt)
// 绕 Y 轴旋转
x2 = x*cos(rot) - z1*sin(rot)
z2 = x*sin(rot) + z1*cos(rot)
// 最终位置（星云整体放在 z=-120 处）
position = (x2, y1*0.4, z2 - 120)
```

### 2.3 触发出口

- **点击任意节点** → 进入飞渡态，飞向该节点
- **搜索框输入节点名点跃迁** → 进入飞渡态，飞向搜索结果节点
- 用户此时可 OrbitControls 自由旋转视角（可选，初始默认锁定更有仪式感）

## 三、状态二：飞渡过渡态

### 3.1 相机动画

相机从当前位置平滑飞到焦点正前方：
- **目标位置**：焦点世界坐标 + 当前视线方向退后 40 单位 + 稍微抬高 (0, 3, 0)
- **目标看向点**：焦点世界坐标
- **缓动**：`easeOutCubic`（快速启动，缓慢停下）
- **时长**：0.8 ~ 1.2 秒
- 动画期间禁用 OrbitControls（避免操作冲突）

### 3.2 节点布局动画

所有节点从星云位置 lerp 到焦点布局位置，不同角色有**不同延迟和弹性**，形成「被拽着走」的层次感：

| 角色 | 延迟 | 动画 | 时长 | 感觉 |
|------|------|------|------|------|
| 焦点 F / 祖先 P | 0ms | easeOutCubic lerp | 800ms | 率先到位 |
| 主链节点 | 50ms | easeOutCubic lerp | 800ms | 紧随焦点 |
| 支链节点 | 150~250ms | easeOutBack 弹性 | 1000ms | 被主链甩出去，超调回弹 |
| 背景节点 | 300ms | opacity 渐变 + lerp | 600ms | 最后淡入，不抢注意力 |

**「被拽着走」效果实现要点**：
- 支链在 delay 期间，不 lerp 到最终目标，而是 copy 对应主链节点的**当前位置**
- delay 结束后，用 easeOutBack 从主链当前位置弹开到目标偏移位
- 叠加微小的随机弹性偏移（overshoot），模拟被甩出的物理感

## 四、状态三：焦点导航态

这是核心交互状态，用户可以自由探索、点击任意节点重新聚焦。

### 4.1 视觉规则

最终形态**恒为同构**：
- 居中上方：祖先节点 P（parent 位）
- 水平排列：主链节点链（居中的就是焦点 F）
- 每个主链节点身后：锥形尾焰散布的支链节点
- 极远处：淡化的背景节点

配图：`cosmos-graph-几何布局_架构图.drawio` 面板①正视图

### 4.2 相机局部坐标系（关键）

布局计算以**点击瞬间**的相机朝向为参考，确保支链环面**平行于屏幕**：

```
forward  = camera.getWorldDirection()   // 相机看向 -Z
up       = camera.up.clone()            // 世界上方向 (0,1,0)
right    = forward.cross(up).normalize() // 右方向（叉积）
back     = forward.clone().negate()     // 远离相机方向
```

**核心原则**：
- 布局瞬间用这三个基向量把局部坐标转为世界坐标
- **布完局后节点固定在世界坐标，不做 billboard**
- 用户自由旋转 OrbitControls 后，侧面看就是真正的3D锥形尾焰
- 这保证了「正看是圆环、侧看是鱼尾」的3D空间感

### 4.3 坐标布局公式

#### 焦点 F 的位置

```
focusWorld = camera.position + forward * (-30)
// 即相机正前方 30 单位处
```

#### 祖先节点 P（parent 位）

```
pPos = focusWorld + up * 10 + back * 10
// 在焦点正上方10、再往远处（-Z方向）10
```

#### 主链节点（横向排列）

```
mainPos[i] = focusWorld + right * (i - centerIdx) * d + back * 10
// d = 主链间距（推荐 5）
// centerIdx 是焦点在主链数组中的索引
```

#### 支链节点（锥形尾焰）——核心算法

**θ 角分配**（支持上下游不对称 m ≠ n）：

```
上游 m 个节点（y>0，屏幕上半环）:
  θ_i = π * (i+1)/(m+1),  i = 0..m-1
  → 均匀分布在 (0, π)，节点间夹角 = π/(m+1)
  → 避免节点出现在 θ=0 和 θ=π（左右水平位），留空

下游 n 个节点（y<0，屏幕下半环）:
  θ_j = π + π*(j+1)/(n+1),  j = 0..n-1
  → 均匀分布在 (π, 2π)
```

**锥形尾焰公式**（B2方案：中心喷射最远 · 鱼尾形）：

```
depth = D_min + (D_max - D_min) * |sin(θ)|
r     = depth * tan(α)

// 其中：
//   D_min = 3~5    （侧面节点的最近深度，紧贴主链）
//   D_max = 15~20  （正上方/正下方节点的最远深度，窜得最远）
//   α     = π/5 ≈ 36°  半锥角

// 关键：|sinθ| 在 θ=π/2 和 3π/2 时 =1（正上下方向最远）
//       在 θ=0 和 π 时 =0（左右方向最近）
//       这就是「鱼尾」效果：正面看是圆环，侧面看上下方向延伸最远
```

**局部偏移 → 世界坐标**：

```
offset = right * (r * cosθ)     // 水平分量
       + up    * (r * sinθ)     // 垂直分量
       + back  * depth          // 深度（远离相机）

branchWorld = parentMainWorld + offset
```

配图：`cosmos-graph-几何布局_架构图.drawio` 面板②侧视图 + 面板③公式

### 4.4 为什么选 B2 方案（|sinθ| 鱼尾形）

| 方案 | 正视效果 | 侧视效果 | 火箭感 |
|------|---------|---------|-------|
| A 平面圆环 | 圆形 | 一条线 | ❌ 无尾焰感 |
| B1 均匀锥 | 圆形 | 等腰三角 | ⚠️ 太规整 |
| **B2 \|sinθ\| 鱼尾** | 圆形 | **鱼尾形（上下延伸）** | ✅ 喷射感最强 |
| C 柱面错落 | 椭圆 | 长方形 | ❌ 无喷射感 |

B2 方案的侧视图是鱼尾形——正上方/正下方的支链窜得最远，侧面看像火箭尾焰喷射，最符合你的比喻。

## 五、焦点重分类（任意节点都可成为焦点）

这是系统的图导航能力。核心逻辑：**不管点击什么类型的节点，最终形态都是同构的**（横向主链 + 锥形尾焰）。

配图：`cosmos-graph-几何布局_架构图.drawio` 底部重分类规则区

### 5.1 边类型标记（前置条件）

每条边必须预先标记三种类型：

| 边类型 | 标记 | 含义 | 示例 |
|--------|------|------|------|
| parent 边 | ↑ | 层级归属（产业→主链，主链→支链父节点） | a0 → b3 |
| sibling 边 | ↔ | 主链之间的横向关系 | b0-b1-b2-...-b6 |
| child 边 | ↓ | 主链到支链，标注 upstream/downstream | b1 → c0(上游), c3(下游) |

### 5.2 三种焦点场景

**场景一：点击产业节点**
- P = 该产业节点（放在 parent 位）
- 主链 = 沿 sibling 边双向BFS（产业节点连接的主链节点开始找），中心主链节点居中
- 这是初始默认视图

**场景二：点击主链节点**
- F = 该主链节点（焦点居中）
- P = 沿 parent 边向上找到的祖先产业节点（放在 parent 位）
- 主链 = 从 F 沿 sibling 边双向BFS延伸（左右能延多长延多长，端点侧为空也可接受，如点 b0 时右边有 b1~b6，左边为空）
- F 的 child 支链 → 挂在 F 身后做尾焰
- 其他主链的支链 → 各自主链位置身后的尾焰

**场景三：点击支链节点**
- F = 该支链节点（焦点居中）
- P = 该节点的直接父主链节点（上移到 parent 位）
- 主链 = P 的 sibling 链，但以 F 为中心重构：
  - F 替换原居中位置
  - P 移到上方 parent 位
  - 原 sibling 链的其他节点排在 F 左右两侧
- F 的下游/上游 child 节点（如果有的话，即更深层支链）→ F 身后的尾焰

### 5.3 通用布局算法（7步）

```
① 识别被点击节点 F
② 沿 parent 边向上找到 P（祖先节点）
③ 从 F/P 沿 sibling 边双向 BFS，收集主链数组，确定 centerIdx
④ 以点击瞬间 camera 的 right/up/back 基向量，计算 F、P、主链每个节点的目标世界坐标
⑤ 遍历主链每个节点的 child 边，按 upstream/downstream 分配 θ，用尾焰公式计算支链坐标
⑥ BFS 收集与 F 距离 ≥ 2 跳的节点，按距离降级为背景（见下文）
⑦ 启动飞渡动画（相机飞行 + 节点 lerp + 支链弹性）
```

## 六、隔跳节点（背景节点）处理

≥2 跳的节点不作为主结构展示，降级为背景：

| 距离 | 处理方式 | 距离参数 | 外观 |
|------|---------|---------|------|
| 0 跳（F） | 焦点 | 0 | 白色高亮球 + 光晕 |
| 1 跳（P、主链、支链） | 主结构 | 0~30 | 正常球体+光晕+标签+连线 |
| 2 跳 | 近背景 | ~120 | 小球，opacity≈0.15，连线 opacity≈0.05 |
| ≥3 跳 | 远背景 | ~300 | 固定大小 Points（不随距离缩放），几乎不可见 |

**性能优化**：远背景节点用 InstancedMesh 或合并为一个 Points 几何体，一个 draw call 渲染全部。

**探索感关键**：保留这些极淡节点是故意的——它们暗示「这个宇宙还很大，你看到的只是当前焦点的邻域」。用户旋转视角/飞向远处时，LOD 会自动加载这些节点的细节。

## 七、LOD 层级策略

按相机到节点的距离自动切换渲染精度：

配图：`cosmos-graph-动画LOD_架构图.drawio` 面板②

| LOD 级别 | 距离范围 | 渲染方式 | 连线 | 标签 |
|----------|---------|---------|------|------|
| LOD 0 近景 | dist < 50 | 高面数球体 + 光晕 Sprite | opacity=0.4 | 显示 |
| LOD 1 中景 | 50 ≤ dist < 150 | 低面数球体，无光晕 | opacity=0.15 | 隐藏 |

**渐进加载**：当用户旋转相机或飞近某片区域时，原来 LOD2 的点会自动升级为 LOD1/LOD0，形成「飞向星辰、星辰变清晰」的探索体验。

## 八、相机交互规则

| 场景 | 节点行为 | 相机行为 |
|------|---------|---------|
| 星云态 | 椭圆公转 | 静止（可选 OrbitControls） |
| 飞渡态 | lerp 到目标位 | easeOutCubic 飞行，禁用操作 |
| 焦点态·自由旋转 | 固定世界坐标 | OrbitControls 自由旋转/缩放 |
| 焦点态·点击其他节点 | 重新走7步布局 | 飞行到新焦点正对位 |
| 焦点态·点击当前祖先 | **不重排** | TWEEN 飞回正对预设位（0.8s） |
| 焦点态·搜索跳转 | 重新走7步布局 | 飞行到搜索目标 |

**关键设计**：
- 再次点击当前视图的祖先节点（比如正在看 b1 为焦点，再点 a0）→ 不重排节点，只把相机 TWEEN 回 a0 为焦点的预设视角
- 点击其他节点（非祖先、非同焦点）→ 重排所有节点，走完整飞渡流程
- 布局完成后节点固定世界坐标，不做 billboard，旋转看侧面可见真实3D锥形
